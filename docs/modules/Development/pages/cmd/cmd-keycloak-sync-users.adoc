=== Keycloak Gebruikers Synchroniseren

Dit commando synchroniseert bestaande CRM-gebruikers naar Keycloak. Het is vooral handig bij migraties of wanneer je bestaande gebruikers in bulk wilt provisionen in Keycloak.

==== Configuratie

===== Vereiste .env variabelen

Zorg dat de volgende variabelen in `.env` staan:

[source,env]
----
KEYCLOAK_BASE_URL=https://keycloak.local.privatescan.nl
KEYCLOAK_INTERNAL_BASE_URL=http://keycloak:8080
KEYCLOAK_REALM=privatescan

KEYCLOAK_ADMIN_USERNAME=admin
KEYCLOAK_ADMIN_PASSWORD=admin

KEYCLOAK_CLIENT_ID=crm-app
CRM_KEYCLOAK_CLIENT_SECRET=your_client_secret
----

Daarnaast moeten CRM-gebruikers in de database aanwezig zijn (bijv. via seeders of import).

==== Gebruik

===== Dry-run (geen echte wijzigingen)

[source,bash]
----
# Bekijk wat er gesynchroniseerd zou worden, zonder echt naar Keycloak te schrijven
sail artisan keycloak:sync-users --dry-run
----

===== Echte synchronisatie

[source,bash]
----
# Synchroniseer alle in aanmerking komende gebruikers naar Keycloak
sail artisan keycloak:sync-users
----

Het commando:

- synchroniseert eerst de Keycloak-configuratie (realm/clients) via `KeycloakConfigService`;
- logt in als Keycloak admin;
- loopt over alle gebruikers en:
  - maakt nieuwe Keycloak-gebruikers aan waar nodig;
  - slaat gebruikers over waarvan geen (bruikbaar) wachtwoord beschikbaar is;
  - logt aantallen gesynchroniseerd/overgeslagen/fouten.

===== Voorbeeld output

[source,text]
----
Keycloak configuratie synchroniseren...
Authenticeren met Keycloak admin...
=== Samenvatting ===
Gesynchroniseerd: 10
Overgeslagen: 2
Fouten: 0

Let op: Gebruikers met gehashte wachtwoorden kunnen niet worden gesynchroniseerd.
Gebruikers worden automatisch gesynchroniseerd wanneer:
  - Ze worden aangemaakt/bijgewerkt via UserObserver (plaintext wachtwoord beschikbaar)
  - Ze inloggen via SSO (account wordt automatisch gekoppeld)
----

==== Parameters

[cols="1,3,1"]
|===
|Parameter |Beschrijving |Standaard

|`--dry-run`
|Toon wat er gesynchroniseerd zou worden zonder echt naar Keycloak te schrijven.
|`false`
|===

==== Troubleshooting

===== Geen gebruikers gesynchroniseerd

*Mogelijke oorzaken:*

1. Er zijn geen gebruikers met een bruikbaar wachtwoord (alleen gehashte wachtwoorden).
2. Gebruikers zijn al eerder gesynchroniseerd en worden nu overgeslagen.

*Oplossing:*

1. Maak nieuwe testgebruikers aan in de CRM (met bekend wachtwoord) en draai het commando opnieuw.
2. Gebruik login/SSO om accounts automatisch te laten koppelen.

===== Authenticatie naar Keycloak faalt

*Oplossing:*

1. Controleer `KEYCLOAK_ADMIN_USERNAME` en `KEYCLOAK_ADMIN_PASSWORD`.
2. Controleer of Keycloak bereikbaar is vanaf de CRM-container (`KEYCLOAK_INTERNAL_BASE_URL`).
