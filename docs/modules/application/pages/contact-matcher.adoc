=== Contact Matcher Scoring Systeem

==== Overzicht

De contact matcher functionaliteit is uitgebreid met een intelligent scoring systeem dat meerdere contactvelden evalueert om percentage-gebaseerde match scores te bieden. Het systeem helpt gebruikers bij het identificeren van de meest relevante contactmatches bij het werken met leads, met een sterke focus op naamveld matching.

==== Scoring Algoritme

===== Matching Velden en Gewichten

* **Naamvelden**: 90% gewicht totaal (primaire focus)
** first_name, last_name, lastname_prefix, married_name, married_name_prefix, initials
* **E-mailadressen**: 5% gewicht
* **Telefoonnummers**: 5% gewicht

===== Scoring Logica

* **100% match op ALLE naamvelden**: 95% totale score
* **100% match op BELANGRIJKE naamvelden** (first_name, last_name, lastname_prefix): 80% totale score
* **Gedeeltelijke naammatches**: Geschaald tussen 0-80% gebaseerd op match ratio
* **E-mail Matching**: Ondersteunt meerdere e-mails, scores gebaseerd op proportie van matchende e-mails
* **Telefoon Matching**: Normaliseert telefoonnummers (handelt Nederlandse +31 formaat af) en ondersteunt meerdere nummers
* **Maximaal mogelijke score**: 100% (wanneer alle velden perfect matchen)

===== Performance Optimalisatie

Het systeem gebruikt intelligente database pre-filtering om performance te verbeteren:

* Pre-filtert personen met LIKE queries op first_name en last_name
* Includeert married_name in de OR conditie voor bredere matching
* Beperkt initiële query tot 30 potentiële matches voor scoring
* Alleen personen met scores > 0 worden teruggegeven naar de frontend

====== Scoring Query Logica

[source,php]
----
$persons = Person::query()
    ->where('first_name', 'like', '%' . $lead->first_name . '%')
    ->where(function ($query) use ($lead) {
        $query->where('last_name', 'like', '%' . $lead->last_name . '%')
            ->orWhere('married_name', 'like', '%' . $lead->married_name . '%');
    })
    ->limit(30)
    ->get();
----

===== Voor Ontwikkelaars

Het scoring algoritme kan eenvoudig worden aangepast door de gewichten in de `calculateMatchScore()` methode te wijzigen:

[source,php]
----
// Huidige gewichten (bijgewerkt)
$nameWeight = 0.90;      // 90% voor alle naamvelden gecombineerd
$emailWeight = 0.05;     // 5% voor e-mail matching
$phoneWeight = 0.05;     // 5% voor telefoon matching

// Naamveld scoring drempelwaarden
$allNameFieldsMatch = 0.95;      // 95% wanneer alle naamvelden matchen
$importantNameFieldsMatch = 0.80; // 80% wanneer belangrijke velden matchen
----

**Geëvalueerde Naamvelden:**

* `first_name`, `last_name`, `lastname_prefix` (belangrijke velden)
* `married_name`, `married_name_prefix`, `initials` (aanvullende velden)

**Database Pre-filtering:**

* Gebruikt LIKE queries voor performance optimalisatie
* Filtert op first_name en (last_name OF married_name)
* Beperkt tot 30 potentiële matches voor scoring

===== Tips voor Betere Matches

* **Volledige namen**: Zorg ervoor dat leads volledige voor- en achternamen hebben
* **Consistente spelling**: Gebruik consistente spelling van namen
* **E-mail adressen**: E-mail adressen geven sterke validatie van matches
* **Telefoonnummers**: Nederlandse nummers worden automatisch genormaliseerd (+31 → 0)
* **Getrouwde namen**: Het systeem herkent zowel meisjesnamen als getrouwde namen
