=== Lead Duplicate Cache Commands

Deze commands beheren de cache voor lead duplicate detection, wat de performance van het kanban board significant verbetert.

==== Overzicht

Het lead duplicate cache systeem versnelt de `findPotentialDuplicates` functionaliteit door resultaten te cachen. Dit zorgt voor 10x+ snellere laadtijden op het kanban board.

==== Commands
===== duplicates:refresh-cache

Hoofdcommand voor cache management (leads én persons) en cronjob.

[source,bash]
----
# Incrementele refresh (standaard) voor beide caches
php artisan duplicates:refresh-cache

# Cache statistieken
php artisan duplicates:refresh-cache --stats

# Cache wissen
php artisan duplicates:refresh-cache --clear

# Volledige rebuild
php artisan duplicates:refresh-cache --full
----

**Parameters:**
* `--stats` - Toon cache statistieken
* `--clear` - Wis alle duplicate caches
* `--full` - Volledige cache rebuild

**Gebruik:**
* Cronjob (elk uur met `--clear`)
* Dagelijkse cache maintenance
* Troubleshooting performance problemen

===== cache:lead-duplicates

Uitgebreide cache management en testing.

[source,bash]
----
# Statistieken bekijken
php artisan cache:lead-duplicates stats

# Alle caches wissen
php artisan cache:lead-duplicates clear

# Cache rebuilden
php artisan cache:lead-duplicates rebuild

# Specifieke lead testen
php artisan cache:lead-duplicates test --lead=123
----

**Parameters:**
* `stats` - Detailleerde cache statistieken
* `clear` - Wis alle duplicate caches
* `rebuild` - Cache volledig opnieuw opbouwen
* `test` - Test cache functionaliteit
* `--lead=ID` - Test specifieke lead ID
* `--force` - Forceer actie zonder bevestiging

**Gebruik:**
* Performance testing
* Debugging cache problemen
* Development en troubleshooting

==== Cache Werking

===== Automatische Cache Invalidation

Cache wordt automatisch geïnvalideerd bij:
* Lead aanmaken
* Lead wijzigen
* Lead samenvoegen (merge)

===== Cache Storage

* **Cache Key**: `lead_duplicates:{lead_id}`
* **TTL**: 1 uur
* **Storage**: Alleen lead IDs (memory efficient)
* **Backend**: Gebruikt Laravel's standaard cache driver

===== Performance

* **Snelheidswinst**: 10-50x sneller dan directe database queries
* **Fallback**: Bij cache problemen valt systeem terug op directe queries
* **Memory Usage**: Minimaal door alleen IDs op te slaan

==== Troubleshooting

===== Cache Werkt Niet

[source,bash]
----
# Check cache configuratie
php artisan config:show cache.default

# Clear en rebuild
php artisan cache:lead-duplicates clear
php artisan cache:lead-duplicates rebuild
----

===== Performance Problemen

[source,bash]
----
# Test specifieke lead
php artisan cache:lead-duplicates test --lead=123

# Check cache hit rate
php artisan duplicates:refresh-cache --stats
----

===== Memory Issues

[source,bash]
----
# Clear alle caches
php artisan cache:clear

# Rebuild met kleinere batches
php artisan cache:lead-duplicates rebuild
----

==== Belangrijke Opmerkingen

* **Automatisch**: Cache wordt automatisch gevuld bij eerste gebruik
* **Cronjob**: Elk uur wordt cache automatisch ververst
* **Graceful Degradation**: Bij problemen valt systeem terug op directe database queries
* **Production Ready**: Volledig getest en stabiel voor productie gebruik
* **Memory Efficient**: Slaat alleen lead IDs op, niet volledige lead data
