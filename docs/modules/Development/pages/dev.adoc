=== Migratie script opschonen

Na verloop van tijd zal het aantal migratie files groeien. Dit kan ervoor zorgen dat het opzetten van een nieuwe omgeving lang duurt. Laravel biedt de mogelijkheid om de huidige database structuur te dumpen naar een sql file en alle migratie files te verwijderen.

1. run sail artisan schema:dump (zonder --prune)
2. doe git add op de nieuw dump file (zie in database de sql file).
3. Voeg deze toe aan Git

Normaal gesproken zou je liever met de optie --prune willen draaien, zodat de migratiescripts ook worden verwijderd.
In dit geval draaien de testcases echter op SQLite, omdat dit de uitvoering van de tests aanzienlijk versnelt.
Het nadeel daarvan is dat SQLite niet werkt met de database dump file.
Daarom gebruiken de testcases in dit scenario wél de migratiescripts.

=== Migratie instructies

==== Samenvatting

Je dient een mysql dump te maken vanuit de oude server, export met Filezilla te downloaden naar de Citrix omgeving, dan kopiëren van Citrix omgeving naar lokaal. Van lokaal met scp naar de nieuwe server en dan importeren in de mysql docker.

Beter is rechtstreeks van de oude server naar de nieuwe met scp, maar er zijn nog firewall issues.

==== Stappen plan om een nieuwe prod suite CRM dataset klaar te zetten

1. **MySQL dump vanuit SSH sessie oude server**
   - Run `./export_...sh` (note: bestand staat in /home/mbulthuis)

2. **Copy file naar ./tmp op de server**

3. **Open Filezilla en copy file naar Citrix omgeving**

4. **Copy naar lokaal**
   - Voor Mac: werkt alleen als je in Safari zit met plugin

5. **SCP naar nieuwe server iChosting**
   - `scp [filename] mbulthuis@178.251.28.76:/home/mbulthuis`

6. **Uitpakken**
   - `gunzip -c scrm_privatesuite9_2025-09-02_1620.sql.gz > scrm_privatesuite9_2025-09-02_1620.sql`

7. **Database aanmaken** [optioneel, alleen eerste keer]
   - https://phpmyadmin.dev.privatescan.nl -> maak database aan met naam `sugarcrm_privatesuite9`

8. **Importeren in MySQL docker**
   - `sudo docker exec -i mysql mysql -u root -p[DB_PASSWORD] scrm_privatesuite9 -h 127.0.0.1 -P 3306 < ./scrm_privatesuite9_2025-09-02_1620.sql`
(DB_PASSWORD is te vinden in .env.prod DB_PASSWORD)

9. (optional) Copy file to local machine

=== Server installatie

==== Github docker login
Eerste keer docker login uitvoeren:
sudo echo <token> | sudo docker login ghcr.io -u <username> --password-stdin

Eerste keer een docker netwerk aanmaken

==== Firewall
- Poort 80 dient open te staan voor traefik (aanmaken van self-signed certificaten)
- Poort 443 dient open te staan privatescan saas

==== Synchronisatie n8n workflows

De eerste keer dien je een gebruiker te registeren.
1. Zet project id in .env file (N8N_PROJECT_ID). Je kunt deze vinden in mysql met select * from project;
2. Maak een API key aan bij je gebruiker en plaats deze in .env file (N8N_API_KEY).


===== Troubleshooting
 *Synchroniseren forceren*
Verwijderen alle bestaande workflows voordat je opnieuw gaat synchroniseren. Je kunt dit doen met een mysql connectie:
mysql -u <DB_MYSQLDB_USER> -p'<DB_MYSQLDB_PASSWORD>' -h 127.0.0.1 -P 3307 <DB_MYSQLDB_DATABASE>
delete from workflow_entity;
delete from shared_workflow;



=== Mail server lokaal opzetten

Het idee is om een eenvoudige mail server lokaal op te zetten. Later gaan we dit met bestaande mail accounts configureren en mail versturen via PostMark.

In de opzet maken wij 2 mail accounts aan:

 * user1@test.local -> Hier mail naar toe sturen vanuit CRM, simuleert een patient.
 * crm@privatescan.nl -> CRM leest deze mailbox en zal ze verwerken na en commando (Lees later een cronjob om de 5 minuten)

test.local is een gekozen opzet, certificaten zijn hiervoor gemaakt voor communicatie met de mailserver.

zie mailserver in docker-compose.yml.

NOTE: Op dit moment is deze opzet ook uitgerold op de test omgeving.

1. User aanmaken in de mailserver (mag je skippen, werkt out of de box door config file te delen. Zie postfix-accounts.cf):
   * docker run --rm -v "$(pwd)/docker/config":/tmp/docker-mailserver \
      mailserver/docker-mailserver setup email add user1@test.local test1234
   * docker run --rm -v "$(pwd)/docker/config":/tmp/docker-mailserver \
      mailserver/docker-mailserver setup email add crm@privatescan.nl test1234
2. rainbow config aanpassen (na opstarten):
    * http://localhost:8888/?admin
    * Inloggen met admin / 12345

image::rainbow_config.png[]

3 CRM mail binnen halen is (voor nu) handmatig te doen. Door dit commmando te draaien:

----
sail artisan inbound-emails:process
----

Note: Op de test omgeving is dit geautomatiseerd via een cronjob die elke minuut draait.

==== Cronjobs

Het systeem heeft verschillende geautomatiseerde taken die via Laravel's scheduler draaien:

===== Actieve Cronjobs

* **`inbound-emails:process`** - Elke 5 minuten
- Verwerkt inkomende emails vanuit de mailbox
- Zet emails om naar activities en leads

* **`activities:release-overdue`** - Elk uur
- Maakt achterstallige activities vrij voor andere gebruikers
- Voorkomt dat taken vast blijven zitten bij gebruikers

* **`duplicates:refresh-cache --clear`** - Elk uur
- Ververst de lead en person duplicate detection cache
- Zorgt voor optimale performance van het kanban board en person overzicht
- Automatische fallback naar directe queries bij problemen

===== Scheduler Status

[source,bash]
----
# Bekijk alle geplande taken
php artisan schedule:list

# Test scheduler handmatig
php artisan schedule:run

# Specifieke taak uitvoeren
php artisan duplicates:refresh-cache --clear
