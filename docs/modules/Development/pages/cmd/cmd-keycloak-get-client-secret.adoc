=== Keycloak Client Secret Ophalen

Dit commando haalt de client secret op voor een Keycloak client en toont direct de juiste `ENV`-sleutel om in je `.env` bestand te plaatsen.

==== Configuratie

===== Vereiste .env variabelen

Voor het ophalen van een client secret moet de basis Keycloak-config geconfigureerd zijn:

[source,env]
----
KEYCLOAK_BASE_URL=https://keycloak.local.privatescan.nl
KEYCLOAK_INTERNAL_BASE_URL=http://keycloak:8080
KEYCLOAK_REALM=privatescan

KEYCLOAK_ADMIN_USERNAME=admin
KEYCLOAK_ADMIN_PASSWORD=admin

# Standaard client ID (optioneel, kan ook als argument)
KEYCLOAK_CLIENT_ID=crm-app
----

Als je geen `client_id` argument meegeeft, wordt de standaard client ID uit de configuratie gebruikt.

==== Gebruik

===== Client secret voor CRM client

[source,bash]
----
# Gebruik standaard client ID uit config (.env)
sail artisan keycloak:get-client-secret
----

===== Client secret voor specifieke client

[source,bash]
----
# Client secret voor forms-app ophalen
sail artisan keycloak:get-client-secret forms-app
----

===== Voorbeeld output

[source,text]
----
Ophalen van client secret...
✓ Client secret voor crm-app:
   <secret-value>

⚠ Voeg dit toe aan je .env file:
   CRM_KEYCLOAK_CLIENT_SECRET=<secret-value>
----

==== Parameters

[cols="1,3,1"]
|===
|Parameter |Beschrijving |Verplicht

|`client_id`
|Client ID waarvan je de secret wilt ophalen. Laat leeg om de standaard client uit de config te gebruiken.
|Nee
|===

==== Troubleshooting

===== Client niet gevonden

[source,text]
----
Kon client secret niet ophalen.
----

*Oplossing:*

1. Controleer of de client bestaat in de opgegeven realm.
2. Controleer of je de juiste `client_id` hebt gebruikt (exacte naam).

===== Geen toegang tot client secret

Als de Keycloak user geen rechten heeft om de clientconfig te lezen, kan het ophalen falen.

*Oplossing:*

1. Zorg dat de admin-gebruiker de juiste rollen heeft (bijv. `realm-admin`).
2. Controleer of de gebruikte Keycloak Admin API-URL (`KEYCLOAK_INTERNAL_BASE_URL`) correct is.
