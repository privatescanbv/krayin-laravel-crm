=== SugarCRM Lead Import

Deze functie importeert leads uit een SugarCRM database naar Krayin CRM, inclusief anamnese gegevens, call activities, emails, email bijlagen en pipeline mapping.

==== Configuratie

===== Database Connectie

Voeg de volgende variabelen toe aan je `.env` bestand:

[source,env]
----
# SugarCRM Database Connectie
SUGARCRM_DB_HOST=127.0.0.1
SUGARCRM_DB_PORT=3306
SUGARCRM_DB_DATABASE=sugarcrm
SUGARCRM_DB_USERNAME=your_username
SUGARCRM_DB_PASSWORD=your_password
SUGARCRM_DB_PREFIX=
----

De import functie verwacht de volgende tabellen in de SugarCRM database:

**Email Attachments tabellen (optioneel):**

* `notes` - Email bijlagen tabel
** `id` - Unieke ID
** `name` - Bijlage naam/titel
** `filename` - Bestandsnaam
** `file_mime_type` - MIME type van bestand
** `description` - Beschrijving van bijlage
** `parent_id` - Koppeling naar emails.id
** `parent_type` - Moet 'Emails' zijn

**Email adressen tabellen (optioneel voor lead email adressen):**

* `email_addr_bean_rel` - Email koppeling

**Call Activities tabellen (optioneel):**

==== Pipeline Mapping

Alle leads worden geïmporteerd naar de **Privatescan pipeline (ID: 1)**.

===== Stage Mapping

[cols="2,3,1"]
|===
|SugarCRM Workflow Status |Krayin Pipeline Stage |Stage ID

|`nieuweaanvraag`, `nieuwe-aanvraag`, `kwalificeren`
|Nieuwe aanvraag kwalificeren
|1

|`adviseren`, `klant-adviseren`, `adviseren-start`
|Klant adviseren
|2

|`opvolgen`, `adviseren-opvolgen`
|Klant adviseren opvolgen
|3

|`won`, `converted`
|Won
|4

|`lost`, `dead`
|Lost
|5
|===

**Prioriteit:** Lead status heeft voorrang over workflow status. Als geen mapping wordt gevonden, wordt stage 1 gebruikt.

==== Gebruik

===== Basis Import

[source,bash]
----
# Dry-run - toon wat er geïmporteerd zou worden
sail artisan import:leads --dry-run

# Import 50 leads (standaard: 100)
sail artisan import:leads --limit=50

# Import specifieke lead IDs
sail artisan import:leads --lead-ids="abc123 def456 ghi789"

# Import met andere database connectie
sail artisan import:leads --connection=sugarcrm_prod

# Import personen vóór elke lead-batch (helpt bij person-matching)
sail artisan import:leads --import-persons -vv
----

===== Parameters

[cols="1,3,1"]
|===
|Parameter |Beschrijving |Standaard

|`--connection`
|Database connectie naam
|`sugarcrm`

|`--limit`
|Aantal leads om te importeren
|`100`

|`--lead-ids`
|Specifieke lead IDs om te importeren (negeert limit)
|`-`

|`--dry-run`
|Test de import zonder daadwerkelijk data op te slaan
|`false`

|`--import-persons`
|Importeer gerelateerde personen per batch vóór lead import (gebruikt `import:persons` met spatie-gescheiden `--person-ids` en geeft dezelfde verbosity door)
|`false`
|===

==== Person Matching

De importer zoekt naar bestaande personen in de volgende volgorde:

1. **Email adres** - Exacte match in `emails` veld
2. **Telefoonnummer** - Match in `phones` of `contact_numbers` velden
3. **Voor- en achternaam** - Exacte match van beide velden

**Belangrijk:** Als geen persoon wordt gevonden, wordt de lead **niet** geïmporteerd en wordt een foutmelding getoond.

==== Anamnese Data

Alle anamnese gegevens uit `leads_cstm` worden automatisch geïmporteerd naar de `Anamnesis` tabel:

* **Medische geschiedenis** - Hart, diabetes, roken, rugklachten, etc.
* **Erfelijke aandoeningen** - Hart, vaten, tumoren
* **Operaties en implantaten** - Inclusief opmerkingen
* **Allergieën en medicijnen** - Met details
* **Fysieke gegevens** - Lengte, gewicht
* **Opmerkingen** - Alle `opm_*_c` velden

==== Call Activities

Call activities worden automatisch geïmporteerd als **Activity** records van het type `call`:

===== Status Mapping

[cols="2,2,1"]
|===
|SugarCRM Status |Krayin Status |Is Done

|`held`, `completed`, `done`, `finished`
|Afgerond
|`true`

|Alle andere statussen (bijv. `planned`)
|Niet afgerond
|`false`
|===

===== Geïmporteerde Gegevens

* **Titel** - `calls.name`
* **Beschrijving** - `calls.description` 
* **Start tijd** - `calls.date_start`
* **Eind tijd** - `calls.date_end`
* **Status** - Gemapped naar `is_done` boolean
* **Custom data** - Opgeslagen in JSON veld:
** `external_id` - Originele SugarCRM call ID
** `direction` - Gesprek richting (inbound/outbound)
** `status` - Originele SugarCRM status
** `belgroep` - Bel groep uit `calls_cstm.belgroep_c`

**Belangrijk:** Call activities worden alleen geïmporteerd voor leads die succesvol zijn geïmporteerd. Als een lead wordt overgeslagen, worden de bijbehorende call activities ook overgeslagen.

==== Emails

Emails worden automatisch geïmporteerd als **Email** records in de emails tabel:

===== Status Mapping

[cols="2,2,1"]
|===
|SugarCRM Status |Krayin Status |Is Done

|`sent`, `archived`, `delivered`
|Afgerond
|`true`

|Alle andere statussen (bijv. `draft`, `pending`)
|Niet afgerond
|`false`
|===

===== Geïmporteerde Gegevens

* **Titel** - `emails.name` (email subject)
* **Beschrijving** - Geformatteerde email content:
** Subject: [email onderwerp]
** Body: [email inhoud, HTML tags gestript]
* **Verzend tijd** - `emails.date_sent`
* **Status** - Gemapped naar `is_done` boolean
* **Custom data** - Opgeslagen in JSON veld:
** `message_id` - Unieke message ID
** `email_type` - Email type (out, in, etc.)
** `status` - Originele SugarCRM status
** `flagged` - Of email gemarkeerd is als belangrijk
** `reply_to_status` - Reply status
** `intent` - Email intentie/doel
** `mailbox_id` - Mailbox identificatie
** `parent_type` - Originele parent type
** `parent_id` - Originele parent ID

===== Email Content Verwerking

* **HTML Stripping** - HTML tags worden automatisch verwijderd voor plain text opslag
* **Lengte Beperking** - Email content wordt beperkt tot 1000 karakters om lange comments te voorkomen
* **Content Prioriteit** - HTML content heeft voorrang over plain text, fallback naar description veld
* **Subject Integratie** - Email subject wordt geïntegreerd in de comment voor volledigheid

**Belangrijk:** Email activities worden alleen geïmporteerd voor leads die succesvol zijn geïmporteerd. Als een lead wordt overgeslagen, worden de bijbehorende email activities ook overgeslagen.

==== Email Attachments

Email attachments (bijlagen) worden automatisch geïmporteerd als **File** records gekoppeld aan email activities:

===== Geïmporteerde Gegevens

* **Bestandsnaam** - `notes.filename`
* **Bestandspad** - Gegenereerd pad: `email_attachments/{lead_id}/{email_id}/{attachment_id}_{filename}`
* **Koppeling** - Gekoppeld aan de bijbehorende email activity via `activity_id`
* **Timestamps** - `notes.date_entered` en `notes.date_modified`

===== Bestandspad Structuur

Attachments worden opgeslagen met een gestructureerd pad:
```
email_attachments/
  └── {lead_external_id}/
      └── {email_id}/
          └── {attachment_id}_{safe_filename}
```

* **Veilige bestandsnaam** - Speciale karakters worden vervangen door underscores
* **Unieke identificatie** - Attachment ID voorkomt naamconflicten
* **Hiërarchische structuur** - Georganiseerd per lead en email

===== Vereisten en Beperkingen

* **Email Activity Required** - Attachment kan alleen geïmporteerd worden als de bijbehorende email activity succesvol is geïmporteerd
* **Filename Required** - Alleen notes met een `filename` worden beschouwd als attachments
* **Parent Type** - Alleen notes met `parent_type = 'Emails'` worden geïmporteerd
* **Duplicaat Preventie** - Attachments met dezelfde naam en pad worden overgeslagen

**Belangrijk:** Email attachments worden alleen geïmporteerd voor email activities die succesvol zijn geïmporteerd. De daadwerkelijke bestandsinhoud wordt niet geïmporteerd - alleen de metadata en bestandsreferentie.

==== Belangrijke Opmerkingen

* **Vereiste:** Personen moeten al geïmporteerd zijn voordat leads kunnen worden geïmporteerd
* **Volgorde:** Nieuwste leads worden eerst geïmporteerd (ORDER BY date_entered DESC)
* **Filter:** Alleen leads met `deleted = 0` worden geïmporteerd
* **Duplicaten:** Leads met dezelfde titel worden overgeslagen
* **Pipeline:** Alle leads gaan naar Privatescan pipeline (ID: 1)
* **Anamnese ID:** SugarCRM lead ID wordt gebruikt als anamnese ID
* **Activities:** Call activities, email activities en email attachments worden automatisch geïmporteerd indien tabellen beschikbaar zijn
* **Activity Duplicaten:** Activities met dezelfde `external_id` worden overgeslagen
* **Attachment Dependencies:** Email attachments kunnen alleen geïmporteerd worden als de bijbehorende email activities succesvol zijn geïmporteerd

==== Troubleshooting

===== Veelvoorkomende Problemen

**Database Connectie Fout**
[source,text]
----
SQLSTATE[HY000] [2002] Connection refused
----

*Oplossing:* Controleer of de SugarCRM database toegankelijk is en de credentials correct zijn.

**Person Not Found Errors**
[source,text]
----
Person not found for lead: Jan Jansen (ID: abc-123)
----

*Oplossing:*
1. Importeer eerst alle personen met `import:persons`
2. Controleer of email/telefoon/naam overeenkomt tussen leads en persons
3. Gebruik `--dry-run` om te controleren welke personen niet gevonden worden

**Memory Limit Overschreden**
[source,text]
----
Fatal error: Allowed memory size exhausted
----

*Oplossing:* Gebruik een kleinere `--limit` waarde of verhoog het PHP memory limit.

**Anamnesis Creation Failed**
[source,text]
----
SQLSTATE[23000]: Integrity constraint violation: 1062 Duplicate entry
----

*Oplossing:* SugarCRM lead ID bestaat al in anamnesis tabel. Controleer op duplicaten of gebruik andere ID strategie.

**Email Activities Import Failed**
[source,text]
----
Emails table does not exist in SugarCRM database, skipping email activities import
----

*Oplossing:* Email tabellen (`emails`, `emails_text`, `emails_beans`) bestaan niet in de SugarCRM database. Dit is optioneel - de lead import zal doorgaan zonder email activities.

**Email Content Too Long**
[source,text]
----
Data too long for column 'comment'
----

*Oplossing:* Email content wordt automatisch beperkt tot 1000 karakters. Als dit probleem optreedt, controleer de database schema voor de `comment` kolom lengte.

**Email Attachments Import Failed**
[source,text]
----
Notes table does not exist in SugarCRM database, skipping email attachments import
----

*Oplossing:* Notes tabel bestaat niet in de SugarCRM database. Dit is optioneel - de email import zal doorgaan zonder attachments.

**Attachment Without Email Activity**
[source,text]
----
Email activity not found for attachment attachment-001, email_id: email-123
----

*Oplossing:* De bijbehorende email activity is niet gevonden of niet succesvol geïmporteerd. Controleer of email activities correct worden geïmporteerd voordat attachments geïmporteerd worden.

===== Debug Tips

1. Gebruik altijd `--dry-run` eerst om de data te controleren
2. Start met een kleine `--limit` (bijv. 10) voor testing
3. Controleer de Laravel logs voor gedetailleerde foutmeldingen
4. Verifieer dat alle vereiste tabellen bestaan in SugarCRM database
