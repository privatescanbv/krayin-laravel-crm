=== Local Development Setup

Local development:

Database:
- sail artisan migrate (eerste keer, en met wijzigingen in migraties)
- sail artisan db:seed (eerste keer, en met wijzigingen in seeders. Letop: overschrijft bestaande data!)
- reset database: sail artisan migrate:fresh

Logs:
- https://github.com/opcodesio/log-viewer
- Setup (eenmalig): php artisan log-viewer:publish
- Daarna logs te bekijken met link:http://localhost:8000/log-viewer[http://localhost:8000/log-viewer]

=== Feature testen uitvoeren

- sail artisan migrate:fresh --database=mysql_testing
- sail artisan test

=== Migratie instructies

Zet sugarcrm database op de server, vanuit hier te importeren. (gezien de server nog geen connectie kan maken met de sugarcrm server)

1. maak mysql dump (doe ik via phpstorm)
2. scp sugarcrm_privatesuite9_local-2025_07_23_12_25_32-dump.sql mbulthuis@178.251.28.76:/home/mbulthuis
3. https://phpmyadmin.dev.privatescan.nl ->  maak database aan met naam `sugarcrm_privatesuite9`
4. sudo docker exec -i mysql mysql -u root -p[DB_PASSWORD] scrm_privatesuite9 -h 127.0.0.1 -P 3306 < ./sugarcrm_privatesuite9_local-2025_07_23_12_25_32-dump.sql

=== Server installatie

==== Github docker login
Eerste keer docker login uitvoeren:
echo <token> | docker login ghcr.io -u <username> --password-stdin

Eerste keer een docker netwerk aanmaken

==== Firewall
- Poort 80 dient open te staan voor traefik (aanmaken van self-signed certificaten)
- Poort 443 dient open te staan privatescan saas

==== Synchronisatie n8n workflows

De eerste keer dien je een gebruiker te registeren.
1. Zet project id in .env file (N8N_PROJECT_ID). Je kunt deze vinden in mysql met select * from project;
2. Maak een API key aan bij je gebruiker en plaats deze in .env file (N8N_API_KEY).


===== Troubleshooting
 *Synchroniseren forceren*
Verwijderen alle bestaande workflows voordat je opnieuw gaat synchroniseren. Je kunt dit doen met een mysql connectie:
mysql -u <DB_MYSQLDB_USER> -p'<DB_MYSQLDB_PASSWORD>' -h 127.0.0.1 -P 3307 <DB_MYSQLDB_DATABASE>
delete from workflow_entity;
delete from shared_workflow;



=== Mail server lokaal opzetten

zie mailserver in docker-compose.yml.

1. User aanmaken in de mailserver:
   * docker run --rm -v "$(pwd)/docker/config":/tmp/docker-mailserver \
      mailserver/docker-mailserver setup email add user1@test.local test1234
   * docker run --rm -v "$(pwd)/docker/config":/tmp/docker-mailserver \
      mailserver/docker-mailserver setup email add privatescan@test.local test1234
2. rainbow config aanpassen (na opstarten):
    * http://localhost:8888/?admin

image::rainbow_config.png[]

3 CRM mail binnen halen is (voor nu) handmatig te doen. Door dit commmando te draaien:

----
sail artisan inbound-emails:process
----
