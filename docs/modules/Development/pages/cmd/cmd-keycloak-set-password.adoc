=== Keycloak Gebruikerswachtwoord Instellen

Dit commando stelt (of reset) het wachtwoord van een gebruiker in Keycloak op basis van zijn/haar e-mailadres. Je kunt kiezen of het wachtwoord tijdelijk moet zijn (gebruiker moet het dan bij de eerste login wijzigen).

==== Configuratie

===== Vereiste .env variabelen

De basis Keycloak-configuratie moet ingesteld zijn:

[source,env]
----
KEYCLOAK_BASE_URL=https://keycloak.local.privatescan.nl
KEYCLOAK_INTERNAL_BASE_URL=http://keycloak:8080
KEYCLOAK_REALM=privatescan

KEYCLOAK_ADMIN_USERNAME=admin
KEYCLOAK_ADMIN_PASSWORD=admin
----

De gebruiker moet al in Keycloak bestaan (bijvoorbeeld via `keycloak:sync-users` of SSO-login).

==== Gebruik

===== Wachtwoord permanent instellen

[source,bash]
----
# Stel een vast (niet-tijdelijk) wachtwoord in
sail artisan keycloak:set-password user@example.com "SterkWachtwoord123!"
----

===== Tijdelijk wachtwoord instellen

[source,bash]
----
# Stel een tijdelijk wachtwoord in (moet bij eerste login gewijzigd worden)
sail artisan keycloak:set-password user@example.com "Tijdelijk123!" --temporary
----

===== Voorbeeld output

[source,text]
----
Authenticeren met Keycloak admin...
✓ Wachtwoord ingesteld voor gebruiker: user@example.com
Wachtwoord is ingesteld als tijdelijk - gebruiker moet het bij eerste login wijzigen
----

==== Parameters

[cols="1,3,1"]
|===
|Parameter / Optie |Beschrijving |Verplicht

|`email`
|E-mailadres van de gebruiker in Keycloak / CRM.
|Ja

|`password`
|Nieuw wachtwoord dat voor de gebruiker wordt ingesteld.
|Ja

|`--temporary`
|Markeer het wachtwoord als tijdelijk (gebruiker moet het bij eerste login wijzigen).
|Nee
|===

==== Troubleshooting

===== Gebruiker niet gevonden

[source,text]
----
Kon wachtwoord niet instellen
----

*Oplossing:*

1. Controleer of het e-mailadres exact overeenkomt met het e-mailadres in Keycloak.
2. Verifieer dat de gebruiker eerder is aangemaakt (bijv. via `keycloak:sync-users` of SSO).

===== Geen rechten om wachtwoord aan te passen

*Oplossing:*

1. Zorg dat de gebruikte admin-gebruiker voldoende rechten heeft (bijv. `realm-admin`).
2. Controleer of `KEYCLOAK_ADMIN_USERNAME`/`KEYCLOAK_ADMIN_PASSWORD` correct zijn.
