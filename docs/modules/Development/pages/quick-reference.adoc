==== Dagelijkse Werkwijze Checklist

===== Taak aanmaken
1. ✅ Open Trello board: https://trello.com/b/iONGaeXJ/taken-mm
2. ✅ Maak nieuwe taak in lane:
- backlog -> niet CRM gerelateerd
- backlog CRM -> als het om CRM gaat
- current sprint -> CRM gerelateerd en onderdeel van huidige thema (als leads)
- onhold of notes -> indien geen MVP taak, of als het een notitie is
- No MVP -> indien het geen MVP taak is, maar wel een taak die opgepakt moet worden
- Reject -> Verzoeken die zijn afgewezen (We kiezen ervoor om deze wel vast te leggen, om eventuele discussies later te voorkomen)
3. ✅ Vul titel in formaat: `[APP_NAME] | [Thema] | Titel van de taak`, b.v. `[CRM] | Leads | Lead importeren` of Aankoop | Factuur | parsing error (Note: voor automatisering belangrijk dat het formaat wordt aangehouden)
4. Geef label NO_MVP, indien het geen MVP taak is. Plaats dit
5. Plan deze in met link:https://planyway.com/app?collectionId=5577a9a0-62df-11f0-8bd1-473542ed2965[planyway] (indien nodig, groot onderwerp is)

===== Taak Oppakken
1. ✅ Ga naar Trello board: https://trello.com/b/iONGaeXJ/taken-mm
2. ✅ Kies taak uit backlog / current sprint
3. ✅ Verplaats naar "In Progress" lane
4. ✅ Voeg jezelf toe als member aan de kaart

===== Tijdens Development
1. ✅ Start je lokale development omgeving op met start-dev.sh.
2. ✅ Commit regelmatig met duidelijke messages
3. ✅ Run tests voor elke wijziging: `php artisan test`
4. ✅ Update Trello kaart met voortgang

===== Implementatie/ontwerp taak Afronden
1. ✅ Final testing van functionaliteit
2. ✅ Code review zelf uitvoeren
3. ✅ Verplaats naar "Review" lane in Trello
4. ✅ Voeg reviewer toe aan  (auto op MK gezet)
5. ✅ Beschrijf wat er getest moet worden


===== Taak Afronden
1. ✅ Na opleveren komt deze in review lane in Trello
   - Indien de reviewer akkoord geeft, dan zet je de taak op "completed" of sleep je deze naar lane "Done" (automatisering zorgt ervoor dat altijd beide gebeuren). Voeg opmerking toe wat er bekeken is.
   - Indien de reviewer geen akkoord geeft, dan terug zetten in "In Progress", moet opmerking wat de feedback is.

==== Review Process

===== Als Ontwikkelaar
1. **Voor Review**: Zorg voor complete implementatie
2. **Review Aanvragen**: Verplaats naar "Review" + tag reviewer
3. **Feedback Verwerken**: Bij afkeuring terug naar "In Progress"
4. **Completion**: Bij goedkeuring automatisch naar "Done"

===== Als Reviewer
1. **Code Quality**: Check coding standards
2. **Functionaliteit**: Test volgens requirements
3. **Documentation**: Controleer code comments
4. **Feedback**: Geef duidelijke feedback in Trello

==== Planning Guidelines

===== Maandelijkse Planning
* **Thema-gebaseerd**: Focus op 1-2 hoofdthema's per maand
* **10 Items Max**: Niet meer dan 10 taken per maand plannen
* **Buffer Time**: 20% buffer voor onverwachte taken
* **Review Moment**: Elke maand planning evalueren

===== Capaciteitsplanning
* Verwachte doorlooptijd per thema wordt vooraf ingeschat
* Maandelijkse review van geplande vs werkelijke voortgang
* Bijsturing op basis van werkelijke doorlooptijden
* **8 uur per dag** standaard
* **8/32 uur per week** afhankelijk van beschikbaarheid. Leg toekomstige beschikbaarheid van in de uren sheet.
* **Vakantie/ziekte** vooraf communiceren
* **Overwerk** alleen in uitzonderlijke gevallen

===== Maandelijkse Thema Planning
* Planning wordt gedaan op **thema niveau per maand**
* Onderwerpen worden opgedeeld in **maximaal 10 items per maand**
* Dit biedt voldoende granulariteit voor monitoring en bijsturing

==== Troubleshooting Snelgids

==== Veelvoorkomende Problemen

**Database Connection Error**
```bash
# Check .env configuratie
cat .env | grep DB_

# Test database connectie
php artisan tinker
>>> DB::connection()->getPdo()
```

**Permission Denied**
```bash
# Fix storage permissions
sudo chmod -R 775 storage bootstrap/cache
sudo chown -R $USER:www-data storage bootstrap/cache
```

**Docker Issues**
```bash
# Restart containers
docker-compose down && docker-compose up -d

# Check logs
docker-compose logs crm
```

**Frontend Build Errors**
```bash
# Clear node modules
rm -rf node_modules package-lock.json
npm install

# Rebuild assets
./build.sh (doet een npm & yarn run build op root level en in admin module)
```


==== Best Practices Samenvatting

✅ **DO:**
* Commit vaak met duidelijke messages
* Test voor elke push
* Update Trello regelmatig
* Vraag hulp bij onduidelijkheden
* Documenteer complexe code
* Volg coding standards (Pint)

❌ **DON'T:**
* Push zonder tests
* Vergeet Trello updates
* Werk aan meerdere taken tegelijk
* Hardcode configuratie waarden
* Negeer linting errors
